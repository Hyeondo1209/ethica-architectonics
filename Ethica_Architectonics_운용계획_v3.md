## 1. 완결의 정의

- 에티카 **1부(정리 1~15)** 를 웹에서 탐험 가능한 3D 공간으로 구현. **깊이 우선** — 끝까지 구현된 1부가 얕은 5부보다 압도적으로 강하다.
- **1차 용도가 공모전·전시·공개이므로, 완성도의 정의를 교체한다:** 심사자·관람객은 링크를 열고 2분 안에 판단한다. "정리 15개가 전부 있다"보다 **"관람객의 첫 2분이 강하다"** 가 우선한다. 배포 링크·워크스루 영상·작품 스테이트먼트는 부록이 아니라 **작품의 절반**이다.
- **16~36(사물의 생산)은 이번 방학 범위 외로 확정.** 방학 후 별도 사이클로.
- **데스크톱(마우스+키보드) 전용.** 모바일 대응·사운드는 범위 외.

## 2. 기술 스택 (확정 — 합의 없이 변경 금지)

| 계층 | 선택 | 비고 |

| 런타임 | React Three Fiber (Three.js) + Vite | 1인칭·인스턴싱·거리별 텍스트·절차 생성 |
| 기하 불리언 | three-bvh-csg (`HOLLOW_SUBTRACTION`) | 겹침·개구부는 격자 근사 아닌 정공법 CSG |

## 6. 세션 의례 (v3 개정)

**시작**
1. `git clone --depth 5 https://github.com/Hyeondo1209/ethica-architectonics.git` → **DESIGN.md 제일 먼저** + 주요 소스 정독, `git log` 확인.
2. 노션 최신 일지 1건 확인 → **대조: 클론 HEAD가 그 일지의 '① 오늘 한 일'을 포함하는가?** 일지 제목에 ⚠️UNPUSHED가 있거나 내용이 불포함이면 **로컬 > repo로 간주**하고 사용자에게 최신본/push를 확인.
3. 현재 그림 **리드백** → 어긋남을 시작에 교정한 뒤 작업.

**종료**
1. `DESIGN.md` 갱신 (덧붙이지 말고 **고쳐 쓰기**).
2. 노션 일지 기록: ①오늘 한 일 ②현재 상태 ③다음 할 일 ④열린 결정. 제목은 `YYYY.MM.DD Ethica Architectonics 프로젝트 일지 — <산출물 한 줄>` (같은 날 다중 세션 구분). 코드 변경 세션인데 push 전 종료면 제목에 **⚠️UNPUSHED**.
3. **push 대상 파일 목록**을 사용자에게 명시 → 사용자 Commit→Push (당일 원칙).

**세션 유형**
- **결정 세션** (설계·계획·매핑 — 화면 불필요) vs **코드 세션** (로컬 확인 필요). 한 세션 = 산출물 하나.
- 코드 전달은 파일 분할 이후 **바뀐 모듈 파일만**, 항상 present_files.
