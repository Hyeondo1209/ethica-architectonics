## 0. 완결의 정의 / 방법론 / 역할
- **완결 = 에티카 1부(정리 1~15)의 웹 3D 건축 번역.** 깊이 우선. **마감 2026.07.31(방학 끝) · 1차 용도 공모전·전시·공개** → "첫 2분이 강한가"가 "15개가 다 있는가"보다 우선(운용계획 v3 §1). 16~36은 **이번 방학 범위 외**(확정).
- **방법론:** 마르샬 게루의 발생적-구조적 독해. 신을 *전제*하지 않고, 단일 속성 실체에서 신 개념을 *구성*해 나가는 과정으로 1부를 파악. (들뢰즈 비교, 내들러 보조.)
- **역할:** 철학적 해석의 최종 판단·미적 결정 = **사용자(김현도)**. Claude = 구조화·구현·비판적 검토. **사용자의 독해를 멋대로 바꾸지 않는다.** 손 스케치는 리드백→명시 확인→구현.
- **스택(확정):** React Three Fiber(three.js)+Vite, three-bvh-csg, **배포 Vercel — 연결 완료(2026.07.03, `https://ethica-architectonics.vercel.app`; GitHub push 시 자동 재배포).** **합의 없이 변경 금지.** drei 기각(텍스트 품질 문제 시 `<Text>`만 재론). 블렌더 강등(입구 명판으로 대체, 조건부 승격). 반복·절차 구조는 코드로 생성.

## 1. 못 박은 원칙 (LOCKED — 절대 잊지 말 것)
- ★★ **단일 속성 실체 = 항상 얇고 우아하게 휜(S자 / 탄젠트 곡선) "경선 리브" 그 자체.** 두꺼운 별도 기둥이 아니다. 리브 1개 = 단일 속성 실체 1개. 여럿(72)이 무한 반복을 암시. → 전환 완료(별도 기둥류 삭제됨). *(k **1.25 적용 완료**(1-③A, constants 고정) — ② 기각·④ 예비. **1-③B 상부 구현 완료(2026.07.05)** — 최종 잠금 = 상부 실공간 로컬 보행 재확인, §7.)*
- ★★ **탐험 = 그 경선 리브 *중 하나*(φ=0, `RIB_PHI`) 안의 여정.** 별도 종류의 구조물이 아니라 한 리브의 *역할*일 뿐 → **탐험 리브를 위치·굵기·형태로 차별화하면 안 된다**(전부 하나의 실체; 문은 형태 차별화가 아니라 합의된 기능). 들어갈 굵기는 *공간 스케일업*으로 흡수해 왔음(SCALE=3.5). **레퍼런스급 갈림 홀(내부폭 10~13) 수용도 같은 수 — 하이브리드 스케일로 흡수(③ S4.8·k1.25 확정 적용, 내부폭 12.0 — 1-③A).** 문 = −x면(통로쪽)만 CSG 관통, +x(바깥) 불투명 보존 — 구현 완료(1-③A). 두께 단독 증가는 꼭대기 융합(이웃 간격 < 지름)으로 물리 불가 — 검증됨.
- ★★ **리브는 불투명(opaque)이다.** 여정 전 구간(나선·무릎길·전망·연결 통로)에서 보이는 것은 이 리브 *내부*뿐 → **"무한히 많은 리브" 공개 = 연결 통로 끝 테라스 문(2026.07.03 밤 확정 — 관내 잔류 설계가 '리브 출구=공개' 미결을 해소).** 시야 차단 3중: ① 리브 불투명(관내 여정 격리) ② 통로 외피(접근 중 먼 리브 가림) ③ 연결 통로 밀폐(공개 직전 전실). 상부 이웃 틈 ≈ 리브 지름의 0.4배 — 관 밖으로 뚫리는 순간 옆 리브들이 즉시 시야 점령 = 스포. 관내 잔류가 유일 해법.
- **동선(★2026.07.03 밤 상부 교체):** **지상 정의·공리 방(원점)** → 방 안 나선 → 빛우물 디스크 → **박스 연결부** → **거대 원기둥 통로**(평면 다리 → 플랫폼(정리 자리) → 계단) → **리브 문(RIB_Y)** → 리브 안 나선(1p5·6·7) → **무릎길(관내 보행)** → **갈림**: ↑ 상행 계단 끝 = **1p8 전망**(막다름 — 보어 올려다보기) / ↓ 아치 → **연결 통로**(1p9·10) → **테라스 문 = 1p11 공개** → 테라스(1p12~15). **중앙 출발 → 수평 바깥 → 가장자리에서 위 → 관내 횡단 → 중앙 회귀.** 참(LandingPad)·직선계단(StraightFlight) **폐기 — 코드·상수 제거 완료(1-③B)**.
- **돔 = 내재성('외부 없음').** 정의·공리 방 = **큰 리브 돔 중앙의 지상 돔**(시작점). 정리 1~4 = **거대 원기둥 통로 안의 길**.
- **외피 = 거대 원기둥(벽 + 닫힌 빗면 천장) + 슬랜더 박스 연결부.** 원기둥이 정리 1~4 구간을 감싼다 = **스포 방지** + **공간감**. 정면(+x) 창으로 탐험 리브 #0+이웃 ±2만 보임. ★ 원기둥은 항상 굵게, 연결부만 가늘게. **단면 동결(2026.07.03 밤): 원기둥 크기(COR_R)·천장 높이는 고정, 위치만 돔 스케일 따라 이동(§3).**
- **길 = 평면 다리 + 원형 플랫폼(기둥이 지면까지 받침) + 계단.** 플랫폼 = 정리가 놓이는 자리.
- ★ **세로 요소 장부 = ㉯(형태 구분) 확정(2026.07.05 용어정리 — §2-C).** 지배축: *오직 리브만 얇음·S자·배열·불투명* → 리브(실체)와 절대 혼동 불가. 비-리브 세로(선돌·비석·플랫폼 기둥·폴)는 곧음·고립·접지 공통 + 각자 서명. '리브만 실체' 잠금의 오염 방지 장치. ㉮(통일) 기각.
