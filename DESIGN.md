# Ethica Architectonics — 설계 정본 (DESIGN.md)

> **이 문서는 "지금 설계가 *무엇인가*"의 단일 정본이다.**
> 노션 일지는 "언제 *무엇을 했나*"의 연대기. 이 둘은 분리한다 — 섞으면 연결성이 깨진다.
>
> ## 세션 의례 (반드시 지킬 것)
> - **시작:** ① **이 문서를 제일 먼저 읽는다.** ② 현재 그림(원칙·용어·공간모델·인벤토리·열린결정)을 사용자에게 **리드백**한 뒤 작업을 시작한다(어긋남을 작업 중간이 아니라 *시작에* 잡기 위해). ③ 코드는 `curl`로 읽는다(아래 주소). 노션 최신 일지는 최근 연대기 참고용.
> - **끝:** ① **이 문서를 갱신한다**(덧붙이지 말고 *현재 상태로 고쳐 쓴다*). ② 노션 일지에 그 세션의 연대기를 *풍족히* 남긴다(무엇을·왜·산출물).
> - **코드 읽기는 항상 `curl`.** `web_fetch`는 GitHub raw에서 옛 캐시를 반환한다(검증됨).
> - **repo는 보통 뒤처져 있다.** 사용자는 내 App.jsx를 로컬에 반영해 돌리지만 push는 가끔 한다 → repo를 curl하면 옛 코드일 수 있음. 최신은 항상 직전에 전달한 App.jsx 기준. 사용자가 push할 때 App.jsx + DESIGN.md를 함께 올린다.
> - 멘탈 모델: **코드 = GitHub · 설계상태 = 이 문서 · 연대기 = 노션 일지.**

## 0. 완결의 정의 / 방법론 / 역할
- **완결 = 에티카 1부(정리 1~15)의 웹 3D 건축 번역.** 5부 전체 아님. **깊이 우선**(끝까지 구현된 1부 > 얕은 5부). 16~36(사물의 생산)은 욕심 목표.
- **방법론:** 마르샬 게루의 발생적-구조적 독해. 신을 *전제*하지 않고, 단일 속성 실체에서 신 개념을 *구성*해 나가는 과정으로 1부를 파악. (들뢰즈 비교, 내들러 보조.)
- **역할:** 철학적 해석의 최종 판단·미적 결정 = **사용자(김현도)**. Claude = 구조화·구현·비판적 검토. **사용자의 독해를 멋대로 바꾸지 않는다.**
- **스택(확정):** React Three Fiber(three.js). 주연 에셋만 Blender→glTF. 배포 Vercel. **합의 없이 스택 변경 금지.** 반복·절차적 구조는 코드로 생성(손모델링 X).

## 1. 못 박은 원칙 (LOCKED — 절대 잊지 말 것)
- ★★ **단일 속성 실체 = 항상 얇고 우아하게 휜(S자 / 탄젠트 곡선) "경선 리브" 그 자체.** 두꺼운 별도 기둥이 아니다. 리브 1개 = 단일 속성 실체 1개. 여럿(72)이 무한 반복을 암시. **→ 전환 완료**(별도 반투명 기둥 `ClimbShaft`·두꺼운 `Pillars` 모두 삭제됨).
- ★★ **탐험 = 그 경선 리브 *중 하나*(φ=0, `RIB_PHI`) 안의 나선을 오르는 것.** 별도 종류의 구조물이 아니라 한 리브의 *역할*일 뿐. → **탐험 리브를 위치·굵기·형태로 차별화하면 안 된다**(전부 하나의 실체). 들어갈 굵기는 *공간 전체 스케일업*으로 흡수 → **완료: `SCALE=3.5`.**
- ★★ **리브는 불투명(opaque)이다.** 나선을 오르는 동안엔 그 리브 *내부*만 보인다 → **"무한히 많은 리브" 공개는 등반 *중*이 아니라 꼭대기(테라스) 도착 시**에 일어난다. (등반 중 점진 공개 / 리브 반투명 아이디어는 폐기.) → 시야 차단 장치는 **두 개로 분리**: ① 리브 불투명(등반 격리) ② 통로 외피(접근 중 먼 리브 가림).
- **동선:** **지상 정의·공리 방(큰 돔의 열린 중심=원점)** → 방 안 나선 올라 빛우물 디스크 → **박스 연결부** → **거대 원기둥 통로**(평면 다리 → 원형 플랫폼(정리 자리) → 계단) → 가장자리 탐험 리브 → 리브 안 나선 → 참·직선계단 → 테라스(도착). **중앙에서 출발 → 수평 바깥 → 가장자리에서 위.**
- **돔 = 내재성('외부 없음').** 정의·공리 방 = **큰 리브 돔 중앙의 지상 돔**(시작점; 옛 '지하' 폐기). 정리 1~4 = **거대 원기둥 통로 안의 길**(평면 다리·플랫폼·계단).
- **외피 = 거대 원기둥(벽 + 닫힌 빗면 천장) + 슬랜더 박스 연결부.** 원기둥이 정리 1~4 구간을 감싼다 = **스포 방지**(불투명 벽이 먼 리브 밭 가림) + **공간감**(천장 빗면 70→150 솟음, airtight). **정면(+x) ±30° 창**으로 탐험 리브 #0+이웃 ±2만 보이고 ±3+는 벽이 가림. **방(원점)↔원기둥은 슬랜더 박스 연결부로 이음**(박스 폭=빛우물 실린더 폭). ★ 테이퍼 통로는 공간감 소실로 폐기 — *원기둥은 항상 굵게, 연결부만 가늘게.*
- **길 = 평면 다리(방→플랫폼) + 원형 플랫폼(기둥이 지면까지 받침) + 계단(플랫폼→리브).** 플랫폼 = **정리가 놓이는 자리**(우선 1p1 하나, 추후 1p1~1p4 사슬: 다리-플랫폼-다리…). 평면 다리는 접근하며 앞 시야가 트이고, 계단은 마지막 리브 접합 구간에만 둔다.

## 3. 공간 모델 + 현재 파라미터 (현 코드 = 직전 전달 App.jsx; repo는 뒤처질 수 있음)
- **스케일:** `SCALE=3.5`. 모든 건축 치수에 곱함. 사람 치수는 고정.
- **돔/리브:** `R_BASE=60×S=210`(지표) → `R_TOP=35×S=122.5`(천장 개구부), `H=200×S=700`. 곡선 `rOf(u)`: `KNEE=0.25`(u공간, 고정)에서 `tanh`(`WIDTH=0.02`)로 급히 좁아짐 — **무릎 아래(u<~0.22)는 거의 수직(원통, r≈210)**. `MERIDIANS=72`, 리브 굵기 `SHELL_RIB_R=1.1×S=3.85`(**불투명**, DoubleSide). `RIB_PHI=0`(탐험 리브 각도; 단 모든 리브 동일). **선호 밀도 = 빽빽한 72개.**
- **나선(`RibStair`, φ=0 리브 *안*):** `U_LAND=0.22` → `Y_LAND=H×0.22=154`(나선 끝 높이, 무릎 살짝 아래). `STEP_RISE=0.32`(고정) → `STAIR_STEPS≈481`(자동=높이÷보폭), `STEPS_PER_TURN=30` → 약 16바퀴. `STAIR_R=SHELL_RIB_R×0.55≈2.12`(리브 내부에 들도록 굵기 비례). 디딤판 `TREAD_DEPTH=1.5`/`TREAD_WIDTH=1.0`/`TREAD_THICK=0.20`, 중앙 폴 `POLE_R=0.35`.
- **테라스:** `TERRACE_Y=H×KNEE=175`, `TERRACE_RIN=27×S=94.5`~`TERRACE_ROUT=33×S=115.5`, `TERRACE_ARC=2.4`(+x). 참 `LandingPad`(`PAD_SIZE=6`) + 직선계단 `StraightFlight`(`FLIGHT_LEN` 자동, `FLIGHT_WIDTH=3.5`)로 나선 top(154)→테라스(175) 연결.
- **정점:** `Apex` y=H=700, 광원+구.
- **지상 정의·공리 방(`DefAxiomRoom`):** `ROOM_CX=0`(**큰 리브 돔의 열린 중심=원점**; 옛 42에서 이동). `ROOM_R=26×S=91`(바닥=가장 넓은 반지름), `ROOM_CEIL_Y=14×S=49`(돔 apex=통로 접합 높이, **변경 금지**), `ROOM_FLOOR_Y=0`(지상 — 더는 아래로 안 팜), `ROOM_HEIGHT=49`. **불투명 납작 돔**(sphereGeometry, apex에 오큘러스). 바닥 원판 + 내부 pointLight. **내벽 나선계단(`ROOM_STAIR_*`):** `INSET=0.36`(나선반지름=벽반지름×비율, ↓=벽서 멀어짐), `TURNS=2.5`, `WIDTH=2.4`, `TOPR=5`(꼭대기 코일 최소반지름), `RISE=0.42`→`STEPS≈117`(자동). 속찬 디딤판+스트링거 튜브. **오큘러스 `ROOM_OCULUS=0.066`**(반지름≈6=박스 반폭). **착지 디스크 `ROOM_LAND_R=6`**(y=49)에 나선이 내려섬. **빛우물 실린더:** 디스크 위로 솟는 슬랜더 통(반지름 6, y≈46→`ROOM_CYL_TOP=90`, 위 열림)+꼭대기 down-light — 좁은 시야로 꼭대기 빛만, 벽이 리브 가림(스포). ⚠️ 옛 b버전(지하 깊게 팜·정의/공리 링 마커·반투명 천장)은 **폐기**(코드에 없음).
- **거대 원기둥 + 박스 연결부(`Corridor`):** `COR_Y0=49`(길 바닥=방 apex), `COR_X0=ROOM_CX=0`(전체 안쪽 끝=방), `COR_X1=210`(리브). **★ `COR_CYL_X0=42`**(거대 원기둥 안쪽 시작=박스 연결부 바깥 끝; **원기둥 크기를 방 위치와 분리하는 핵심 노브**, ↓=원기둥 큼). **거대 원기둥:** 중심 `COR_CX=(42+210)/2=126`, 반지름 `COR_R=(210−42)/2=84`(**고정 — 방 위치 무관**), 둘레 `COR_WALL_SEG=96`, 바닥 원이 지면0에 앉음. **빗면 천장:** `CEIL_LO=70`→`CEIL_HI=150`, `ceilY(x)=70+slope·(x−42)`(slope=`COR_CYL_X0` 기준). 벽=바닥0/돔표면→`ceilY` 둘레 띠, 천장=중심→림 삼각부채(닫힘). **정면 창(±2):** `WIN_HALF=asin(42/COR_R)=π/6`(물리 z폭 42 유지→±2 보임/±3 가림; COR_R 바뀌어도 보존), `WIN_SILL_Y=0`~`WIN_TOP_Y=150` 풀하이트. **박스 연결부(방↔원기둥):** `BOX_X0=COR_X0=0`(방 중심)~`BOX_X1=COR_CYL_X0+4=46`(원기둥 안으로 물림), 반폭 `BOX_HW=6`(=빛우물 실린더), 천장 `BOX_TOP=70`, 원기둥 −x 문 `DOOR_HALF=asin(6/84)`. 판 두께 `COR_THICK=0.6`. **방 돔(r91)↔원기둥(−x부 42~91) 겹침**은 `domeClipY`로 벽 밑을 돔 표면까지 잘라 처리. (옛 테이퍼 통로·깔때기 flare/fan·`C`토글 **폐기**.)
- **길(`Corridor` 안):** **평면 다리**(`COR_X0`→`PLAT_X−PLAT_R`, y=49, 상승 0 → 접근 시야 트임) + **원형 플랫폼**(`PLAT_X=(0+210)/2=105`, `PLAT_R=11`, y=49, **정리 자리**) + **받침 기둥**(`PILLAR_R=4`, 플랫폼 밑→지면 0) + **계단**(`PLAT_X+PLAT_R`→`COR_X1`, y=49→`RIB_Y`). 길 반폭 `COR_FLOOR_HW=2.5`(폭 5). 계단(`Stairs`): 칸 높이 `COR_RISE=0.43`(고정), 총 상승 `COR_CLIMB=25`, 칸수 `COR_STEPS=round(25/0.43)=58`(자동), run 73 → 경사≈**18.9°**. `RIB_Y=COR_Y0+COR_CLIMB=74`(리브 접합 높이). 계단은 다리2에만, 다리1은 평면. ⚠️ App.jsx의 `COR_CLIMB` 코멘트는 "≈9.3°"로 옛 값이 남아있음(실제 18.9°) — 사소한 정리 대상.
- **사람 스케일(고정):** 눈높이 `EYE=1.6` 등. **스케일업 시 이 값들은 고정**, 건축 치수만 키운다. 걷기 = `userData{walkable:true}` 표면에 레이캐스트로 높이 맞춤.
- **조작/임시:** `FREE_WALK=true`(벽·낙하 충돌 끔 — 편집용 임시; 끄려면 false). `G`키 돔↔데이터 그래프 뷰. **시작 위치 = 방(원점)**(`ROOM_CX, ROOM_FLOOR_Y+1.6, 0`), +x(리브) 바라봄.

**최근 완료 (이번 세션):**
- **박스 통로 ↔ 원뿔대 ↔ 돔 겹침을 CSG(`three-bvh-csg`)로 정공법 해결.** 면별 클립 근사(격자 컷아웃 등)는 교선이 어긋나 폐기. `HOLLOW_SUBTRACTION`(A 표면 중 B 내부만 제거)으로: 박스 껍질−원뿔대(파고듦 제거)·원뿔대 통벽−박스(통로 구멍)·원뿔대−돔 타원체(구 겹침). **CSG는 렌더 컴포넌트별로 분리해야 함**(변수 스코프): 원뿔대 `wellCut`=`DefAxiomRoom`, 박스 `boxWallCut`/`boxCeilCut`=`Corridor`.

(이전엔 `r=max(TOPR, wallR×INSET)`라 INSET이 max에 먹혀 반지름 불변=원기둥이었고, INSET은 사실상 블록 크기만 조절했음.)
