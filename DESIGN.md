## 0. 완결의 정의 / 방법론 / 역할
- **완결 = 에티카 1부(정리 1~15)의 웹 3D 건축 번역.** 깊이 우선. **마감 2026.07.31(방학 끝) · 1차 용도 공모전·전시·공개** → "첫 2분이 강한가"가 "15개가 다 있는가"보다 우선(운용계획 v3 §1). 16~36은 **이번 방학 범위 외**(확정).
- **방법론:** 마르샬 게루의 발생적-구조적 독해. 신을 *전제*하지 않고, 단일 속성 실체에서 신 개념을 *구성*해 나가는 과정으로 1부를 파악. (들뢰즈 비교, 내들러 보조.)
- **역할:** 철학적 해석의 최종 판단·미적 결정 = **사용자(김현도)**. Claude = 구조화·구현·비판적 검토. **사용자의 독해를 멋대로 바꾸지 않는다.** 손 스케치는 리드백→명시 확인→구현.
- **스택(확정):** React Three Fiber(three.js)+Vite, three-bvh-csg, 배포 Vercel(Phase 1). **합의 없이 변경 금지.** drei 기각(텍스트 품질 문제 시 `<Text>`만 재론). 블렌더 강등(입구 명판으로 대체, 조건부 승격). 반복·절차 구조는 코드로 생성.

## 1. 못 박은 원칙 (LOCKED — 절대 잊지 말 것)
- ★★ **단일 속성 실체 = 항상 얇고 우아하게 휜(S자 / 탄젠트 곡선) "경선 리브" 그 자체.** 두꺼운 별도 기둥이 아니다. 리브 1개 = 단일 속성 실체 1개. 여럿(72)이 무한 반복을 암시. **→ 전환 완료**(별도 반투명 기둥 `ClimbShaft`·두꺼운 `Pillars` 모두 삭제됨).
- ★★ **탐험 = 그 경선 리브 *중 하나*(φ=0, `RIB_PHI`) 안의 나선을 오르는 것.** 별도 종류의 구조물이 아니라 한 리브의 *역할*일 뿐. → **탐험 리브를 위치·굵기·형태로 차별화하면 안 된다**(전부 하나의 실체). 들어갈 굵기는 *공간 전체 스케일업*으로 흡수 → **완료: `SCALE=3.5`.**
- ★★ **리브는 불투명(opaque)이다.** 나선을 오르는 동안엔 그 리브 *내부*만 보인다 → **"무한히 많은 리브" 공개는 등반 *중*이 아니라 꼭대기 도착 시**에 일어난다. (등반 중 점진 공개 / 리브 반투명 아이디어는 폐기.) 시야 차단 장치는 **두 개로 분리**: ① 리브 불투명(등반 격리) ② 통로 외피(접근 중 먼 리브 가림). *(검토 발견 2026.07.02: 공개의 물리적 지점은 참·직선계단이 열린 돔 공간에 있어 사실상 **리브 출구(참 진출) 순간** — "꼭대기 도착"의 해석·정리 배치는 다음 세션에서 확정, §5 미결 포크.)*
- **동선:** **지상 정의·공리 방(큰 돔의 열린 중심=원점)** → 방 안 나선 올라 빛우물 디스크 → **박스 연결부** → **거대 원기둥 통로**(평면 다리 → 원형 플랫폼(정리 자리) → 계단) → 가장자리 탐험 리브 → 리브 안 나선 → 참·직선계단 → 테라스(도착). **중앙에서 출발 → 수평 바깥 → 가장자리에서 위.**
- **돔 = 내재성('외부 없음').** 정의·공리 방 = **큰 리브 돔 중앙의 지상 돔**(시작점; 옛 '지하' 폐기). 정리 1~4 = **거대 원기둥 통로 안의 길**(평면 다리·플랫폼·계단).
- **외피 = 거대 원기둥(벽 + 닫힌 빗면 천장) + 슬랜더 박스 연결부.** 원기둥이 정리 1~4 구간을 감싼다 = **스포 방지** + **공간감**. **정면(+x) 창**으로 탐험 리브 #0+이웃 ±2만 보이고 ±3+는 벽이 가림. ★ 테이퍼 통로는 공간감 소실로 폐기 — *원기둥은 항상 굵게, 연결부만 가늘게.*
- **길 = 평면 다리(방→플랫폼) + 원형 플랫폼(기둥이 지면까지 받침) + 계단(플랫폼→리브).** 플랫폼 = **정리가 놓이는 자리**(현재 1p1 비석, 추후 사슬 여부는 배치 골격에서).

## 3. 공간 모델 — 의도·불변식·핵심 노브 (★수치의 정본 = 코드 주석. 여기엔 '왜'만 남긴다)
- **스케일 불변식:** `SCALE=3.5`는 **건축 치수 전용**. 사람 치수(EYE·계단 보폭 등)는 고정 → 사람만 상대적으로 작아져 리브 안에 들어감. *(검토 지적: `WALK_SPEED=3.5×SCALE`는 이 원칙과 상충 — 속도를 키우면 체감 스케일이 상쇄됨. 릴리즈 전 결정: 사람 속도 고정+달리기 키 vs 현행. §7)*
- **돔/리브:** 곡선 `rOf(u)`=tanh 프로파일, **무릎(KNEE=0.25) 아래는 거의 수직(원통)**. 리브 72개(빽빽한 밀도 선호)·불투명. 탐험 리브는 φ=0이되 형태는 전 리브 동일(LOCKED).
- **리브 나선(`RibStair`):** 리브 중심선을 따라 무릎 살짝 아래(`U_LAND=0.22`)까지. 칸 수 = 높이÷보폭 자동. ⚠️ **현재 u=0(y=0, 지면)에서 시작** — 통로 도착 높이 `RIB_Y`와 불일치 + 리브 튜브에 **출입 개구부 없음** → Phase 1-③에서 CSG 문 + 시작 높이 `RIB_Y` 기준으로 정합(§6).
- **방(`DefAxiomRoom`):** 불변식 — ① `ROOM_CEIL_Y=49`(돔 apex=통로 접합, **변경 금지**) ② 내벽 나선 도착 반지름 `RIN` < 오큘러스 반지름(≈17.45)이어야 관통 ③ 빛우물=**원뿔대**(위 좁음=리브 스포 차단, 바닥=착지 링과 봉합) ④ 겹침(원뿔대−박스−돔)은 **CSG 정공법**(`HOLLOW_SUBTRACTION`), 면별 근사 금지. **내벽 나선 = 공중 낱장 디딤판, 높이 기준 배치**(칸마다 RISE 상승 → 가파른 안쪽 자동 촘촘). **형태 = 원형(매끈) 확정**, 옥타곤은 `T`키 A/B 비교로 보존. ★핵심 상충 원리: 나선이 클수록(ROUT·TURNS↑) 경로가 길어 경사↓·칸 성김 — 계단다움은 **오직 경로길이 축소**로만 얻는다. 현 균형점 = ROUT 45·2바퀴·bias 1.0(고른 상승으로도 헤드룸 통과). **정의·공리 배치(★2026.07.02 확정, 사용자 안):** 정의 옥타곤 = 시작점 둘레 8각(반경·방향 노브 — D3를 +x 후보). 공리 스테이션 = **오르는 시야 왼쪽**이 정본 의도 — 현 감김(ang이 −4π→0 증가, x=cos/z=sin ⇒ 위에서 **시계방향** 상승)에서 왼쪽=**바깥(내벽) 쪽**; 감김을 바꾸면 좌우가 반전되므로 '왼쪽'을 기준으로 유지. 각 스테이션 = 지반(y=0)→그 지점 발높이 기둥+플랫폼+구조물, 경로 균등 7개. ⚠️최상부는 트레드 r≈14라 받침이 원뿔대 기부(r18)·착지 디스크(6~18)와 간섭 → **분포 구간 노브(권고 5~85%)**. 구조물 형태 미정(정리 비석과의 **어휘 분리 여부** = 열린 결정). *바닥 각인안은 1인칭 가시성(원근 눌림·시선 하향)으로 폐기.*
- **통로(`Corridor`):** 불변식 — ① **`COR_CYL_X0`가 원기둥 크기를 방 위치와 분리하는 핵심 노브**(원기둥 반지름 `COR_R`은 방 위치 무관 고정) ② 창 반각 `WIN_HALF=asin(42/COR_R)` = **물리 z폭 42 보존 공식**(±2 보임/±3 가림, `COR_R`이 바뀌어도 유지) ③ 박스 연결부는 원기둥에 살짝 물리고(`BOX_X1=COR_CYL_X0+4`), 박스−원뿔대 겹침은 CSG ④ 방 돔↔원기둥 겹침은 `domeClipY`(벽 밑을 돔 표면까지 클립). 빗면 천장(낮은 방쪽→높은 리브쪽)이 공간감의 핵심.
- **길(★2026.07.02 밤 잠정 채택 — 4플랫폼 슬라럼 B):** 다리(평면) → **플랫폼 ×4(1p1→1p4 읽기 순서, 받침 기둥 각자 지반 직결)** → 계단으로 `RIB_Y`까지. 불변식 — ① 네 플랫폼은 **전부 같은 높이(PLAT_Y, 상승 0)** = 1p1~4의 평행 도출(넷 다 deps에 정리 인용 0, `ethica1.js` 검증). 통로의 첫 상승(계단)은 첫 의존 정리 **1p5**((b) 통로 끝)로 오른다 — *평평함=평행, 상승=의존*. ② 기둥은 진짜 지면에 닿아야 함 → 중심 x ≳ **97**(방 돔 footprint = ROOM_R 91 + 기둥 반지름 + 여유). ③ 이 한계와 계단 각(≤~22°) 사이에 4개 일직선(A안)은 안 들어감(계단 ≈29°) — **B: z 슬라럼(예: z=0/±16)**으로 x 예산 확보 + P2·P3가 창을 비스듬히 보는 자리가 저절로 생김(§5 ②③과 맞물림). 스케치 출발값: R≈7, 중심 x≈104/116/128/140, 계단 시작 ≈147(≈21°) — 확정값 아님, 구현 시 코드 주석이 정본. **B는 잠정** — A(축선) 재론·더 나은 안 교체 여지(사용자 유보).
- **정리 비석(`PropStele`):** 본문은 `ethica1.js`에서 읽음(**텍스트=데이터** 원칙). 슬랩·받침 늘 보임, **글자만** smoothstep 거리 페이드(near/far 노브). 앞면 −x(접근 방향).
- **조작/임시:** `FREE_WALK=true`(편집용 — Phase 1-④에서 끔). 시작 = 방 바닥(원점), +x 바라봄. `G` 그래프, `T` 방 계단 비교.
