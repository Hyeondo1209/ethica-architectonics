## 1. 완결의 정의

- 에티카 **1부(정리 1~15)** 를 웹에서 탐험 가능한 3D 공간으로 구현. **깊이 우선** — 끝까지 구현된 1부가 얕은 5부보다 압도적으로 강하다.
- **1차 용도가 공모전·전시·공개이므로:** 심사자·관람객은 링크를 열고 2분 안에 판단한다. "정리 15개가 전부 있다"보다 **"관람객의 첫 2분이 강하다"** 가 우선한다. 배포 링크·워크스루 영상·작품 스테이트먼트는 부록이 아니라 **작품의 절반**이다.
- **v4 추가:** 조형의 완성도가 첫 2분을 결정한다(친구 "Roblox" 소견 = 조명·렌더링 부재 진단). **조형 확장(P1) + 조명(P2)이 작품의 몸통**, 완주·발표물(P3·P4)이 그것을 전달한다.
- **16~36(사물의 생산)은 범위 외 확정.** 방학 후 별도 사이클.
- **데스크톱(마우스+키보드) 전용.** 모바일 대응·사운드는 범위 외.

## 6. 세션 의례 (v3 계승 — 변경 없음)

**시작**
1. `git clone --depth 5` → **DESIGN.md 제일 먼저** + 주요 소스 정독, `git log` 확인. 클론 실패 시 멈추고 물을 것(기억으로 리드백 금지).
2. 노션 최신 일지(날짜 기준) 확인 → 클론 HEAD가 일지 '① 오늘 한 일'을 포함하는지 대조. ⚠️UNPUSHED거나 불포함이면 로컬>repo로 간주, 사용자 확인. ⚠️UNPUSHED 태그 청소(push 확인 시 제거).
3. 현재 그림 **리드백** → 첫 줄에 '이 세션 = <유형> — 권장 <등급>' 선언 → 작업.

**종료**
1. `DESIGN.md` 갱신(고쳐 쓰기).
2. 노션 일지: ①오늘 한 일 ②현재 상태 ③다음 할 일(각 항목 [A/B급·무게] 태그) ④열린 결정. 제목 `YYYY.MM.DD … — <산출물 한 줄>`, 코드 push 전이면 ⚠️UNPUSHED.
3. push 대상 파일 목록 명시 → 사용자 Commit→Push.

**세션 유형:** 결정 세션 vs 코드 세션. 한 세션 = 산출물 하나. 코드 전달은 바뀐 모듈만, 항상 present_files.
